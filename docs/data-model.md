# Phase 2: Requirement Breakdown & Data Model

This document is the source of truth for the initial MongoDB/Mongoose schema before major feature coding starts.

## Application Modules

- Authentication and User Module: registration, login, role-based access, currency preference.
- Financial Profile Module: income target, savings target, risk preference, financial goals.
- Income and Expense Tracking Module: manual transaction entry, category tracking, monthly summaries.
- Financial Habit Tracking Module: habit setup, completion logging, streaks, completion rate.
- Savings Goal Module: goal creation, progress tracking, completion status.
- Wealth Growth Tracking Module: manual savings, investment, and asset value entries.
- Basic Admin Module: user counts, platform activity, feedback/issues, engagement metrics.

## Collections

### `users`

Purpose: Stores login identity, role, and global user preferences.

Fields:

- `name`: string, required, trimmed.
- `email`: string, required, unique, lowercase, validated email format.
- `passwordHash`: string, required, excluded from queries by default.
- `role`: enum `user | admin`, default `user`.
- `currencyPreference`: enum `INR | USD`, default `INR`.
- `createdAt`: generated by Mongoose timestamps.
- `updatedAt`: generated by Mongoose timestamps.

Indexes:

- Unique email index.
- Role index for admin/user filtering.

### `profiles`

Purpose: Stores financial preferences and long-term planning metadata for each user.

Fields:

- `userId`: ObjectId reference to `users`, required, unique.
- `monthlyIncomeTarget`: number, minimum `0`, default `0`.
- `savingsTarget`: number, minimum `0`, default `0`.
- `riskPreference`: enum `conservative | moderate | aggressive`, default `moderate`.
- `financialGoals`: embedded list of planning goals.
- `createdAt`, `updatedAt`.

Embedded `financialGoals` fields:

- `title`: string, required.
- `targetAmount`: number, minimum `0`.
- `deadline`: date, optional.
- `priority`: enum `low | medium | high`.
- `status`: enum `planned | active | completed | paused`.

Relationship:

- One `users` document has one `profiles` document.

### `transactions`

Purpose: Stores manual income and expense records.

Fields:

- `userId`: ObjectId reference to `users`, required.
- `type`: enum `income | expense`, required.
- `category`: string, required.
- `amount`: number, required, minimum `0`.
- `date`: date, required, default current date.
- `note`: string, optional.
- `createdAt`, `updatedAt`.

Indexes:

- `userId + date` for transaction history.
- `userId + type + category` for dashboard summaries.

Relationship:

- One `users` document has many `transactions`.

### `habits`

Purpose: Stores financial habit definitions such as saving, logging expenses, budget reviews, and investing.

Fields:

- `userId`: ObjectId reference to `users`, required.
- `habitName`: string, required.
- `frequency`: enum `daily | weekly | monthly`, default `daily`.
- `target`: number, minimum `1`, default `1`.
- `streak`: number, minimum `0`, default `0`.
- `status`: enum `active | paused | completed`, default `active`.
- `createdAt`, `updatedAt`.

Indexes:

- `userId + status` for active habit lists.
- `userId + frequency` for reminder/filter views.

Relationship:

- One `users` document has many `habits`.

### `habitLogs`

Purpose: Stores habit completion history.

Fields:

- `habitId`: ObjectId reference to `habits`, required.
- `userId`: ObjectId reference to `users`, required.
- `date`: date, required, default current date.
- `completed`: boolean, default `false`.
- `createdAt`, `updatedAt`.

Indexes:

- `userId + date` for daily/weekly history.
- Unique `habitId + date` to prevent duplicate logs for the same habit/date.

Relationship:

- One `habits` document has many `habitLogs`.
- `userId` is duplicated intentionally for faster user-scoped queries.

### `savingsGoals`

Purpose: Stores specific savings targets and progress.

Fields:

- `userId`: ObjectId reference to `users`, required.
- `title`: string, required.
- `targetAmount`: number, required, minimum `1`.
- `currentAmount`: number, minimum `0`, default `0`.
- `deadline`: date, optional.
- `category`: string, default `General`.
- `status`: enum `active | completed | paused`, default `active`.
- `createdAt`, `updatedAt`.

Indexes:

- `userId + status` for active/completed goal views.
- `userId + deadline` for timeline sorting.

Relationship:

- One `users` document has many `savingsGoals`.

### `assets`

Purpose: Stores manually tracked savings, investments, and asset values for wealth growth analytics.

Fields:

- `userId`: ObjectId reference to `users`, required.
- `name`: string, required.
- `type`: enum `savings | investment | asset`, required.
- `value`: number, required, minimum `0`.
- `date`: date, required, default current date.
- `createdAt`, `updatedAt`.

Indexes:

- `userId + date` for net worth trend charts.
- `userId + type` for asset category breakdown.

Relationship:

- One `users` document has many `assets`.

### `feedback`

Purpose: Stores feedback or complaints for basic admin review.

Fields:

- `userId`: ObjectId reference to `users`, required.
- `message`: string, required.
- `status`: enum `open | in_review | resolved | closed`, default `open`.
- `createdAt`, `updatedAt`.

Indexes:

- `status + createdAt` for admin queue ordering.
- `userId + createdAt` for user feedback history.

Relationship:

- One `users` document has many `feedback` entries.

## Cross-Module Rules

- All user-owned records must include `userId`.
- Amount and value fields must never be negative.
- The default currency is INR through `users.currencyPreference`.
- Bank integration and automatic transaction sync are out of scope for Phase 1.
- Investment entries are manually tracked values only; no trading features are included.
- Admin dashboards should aggregate from existing collections rather than use separate admin-only data.

## Initial Page-to-Collection Mapping

- Login/Register: `users`
- Financial Dashboard: `users`, `transactions`, `habits`, `habitLogs`, `savingsGoals`, `assets`
- Expense Tracker: `transactions`
- Habit Tracker: `habits`, `habitLogs`
- Savings Goals: `savingsGoals`
- Wealth Analytics: `transactions`, `assets`, `savingsGoals`
- Admin Panel: `users`, `transactions`, `habits`, `habitLogs`, `savingsGoals`, `feedback`
